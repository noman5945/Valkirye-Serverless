## EPIC-01 — Identity & Access / Identität & Zugang

> **EN** Everything related to who a user is, how they get in, and what others can see about them.  
> **DE** Alles, was damit zusammenhängt, wer ein Benutzer ist, wie er Zugang erhält und was andere über ihn sehen können.

### Status
`Proposed` — `Vorgeschlagen`

### Summary / Zusammenfassung

**EN**  
Valkirye is a closed, invitation-only platform. Users cannot self-register freely — every account originates from an invitation link generated by an existing registered user. Once registered, a user maintains a profile with a display name and status. Profile details are visible only to accepted friends, preserving privacy by design in alignment with GDPR Article 25.

**DE**  
Valkirye ist eine geschlossene Plattform mit Einladungssystem. Benutzer können sich nicht frei registrieren — jedes Konto entsteht durch einen Einladungslink, der von einem bereits registrierten Benutzer generiert wurde. Nach der Registrierung pflegt ein Benutzer ein Profil mit Anzeigenamen und Status. Profildetails sind nur für akzeptierte Freunde sichtbar, was den Datenschutz gemäß DSGVO Artikel 25 by Design gewährleistet.

---

### Scope / Umfang

#### In Scope / Im Umfang

**Registration & Authentication / Registrierung & Authentifizierung**
- Registration exclusively via a single-use, time-limited invitation link
- Credentials: email address + password
- Authentication via Amazon Cognito User Pools with JWT (access token + refresh token)
- Password reset via verified email
- Account deletion — full data erasure (GDPR Article 17 / DSGVO Artikel 17)
- Secure logout (token invalidation)

**Profile / Profil**
- Display name (publicly visible in invitation and friend-request context only)
- Bio and status message (visible to accepted friends only)
- Email address — private, never exposed via any API response
- Profile update (display name, bio, status)

**Visibility Model / Sichtbarkeitsmodell**
| Field / Feld | Non-friend / Nicht-Freund | Friend / Freund |
|---|---|---|
| Display name | ✓ visible | ✓ visible |
| Status message | ✗ hidden | ✓ visible |
| Bio | ✗ hidden | ✓ visible |
| Email | ✗ hidden | ✗ hidden (always) |
| Friends list | ✗ hidden | ✗ hidden (always) |

**Friends / Freunde**
- Send a friend request by username
- Accept or decline an incoming friend request
- Remove an existing friend
- Friends list — private, never exposed to third parties

**Invitation System / Einladungssystem**
- Any registered user may generate an invitation link
- Invitation links are single-use and expire after 72 hours
- Each invitation carries the sender's `userId` for audit traceability
- Expired or already-used links are rejected with a clear error
- A user may have a maximum of 5 active pending invitations at any time

#### Out of Scope / Außerhalb des Umfangs
- Avatar / profile image upload (deferred to a future story)
- Multi-factor authentication / TOTP (deferred)
- OAuth / social login (Google, GitHub)
- Admin roles or moderation tooling
- Username change after registration

### Actors / Akteure

| Actor / Akteur | Description / Beschreibung |
|---|---|
| **Guest** / Gast | Unauthenticated user holding a valid invitation link |
| **User** / Benutzer | Authenticated registered user |
| **System** / System | Cognito, Lambda, DynamoDB acting autonomously |

---

### Key Constraints / Schlüsselanforderungen

| Constraint / Einschränkung | Rationale |
|---|---|
| Email never returned by API | GDPR data minimisation (Art. 5 §1c) |
| Invitation links single-use | Prevents account sharing and enumeration |
| Friends list always private | GDPR — social graph is personal data |
| Account deletion cascades | GDPR Art. 17 — right to erasure across all tables |
| All data stored in `eu-central-1` | GDPR — data residency within EU |

### High-Level User Stories / Übergeordnete User Stories
> Full stories defined in `user-stories.md`

- `US-01` — As a guest, I can register using a valid invitation link
- `US-02` — As a user, I can log in with my email and password
- `US-03` — As a user, I can reset my password via email
- `US-04` — As a user, I can update my display name, bio, and status
- `US-05` — As a user, I can generate an invitation link to share with someone
- `US-06` — As a user, I can send a friend request by username
- `US-07` — As a user, I can accept or decline a friend request
- `US-08` — As a user, I can remove a friend
- `US-09` — As a user, I can view a friend's full profile
- `US-10` — As a user, I can delete my account and all associated data
